# ⚠️ 已知坑：注册报 500 = passlib 与 bcrypt 4.x 不兼容
#
# 现象：网页注册失败，F12 显示 /auth/register 返回 500。
# 后端日志 traceback 末尾：
#   ValueError: password cannot be longer than 72 bytes
# 原因：passlib 1.7.4 首次 hash() 会用超 72 字节的密码做 bug 探测，
#       bcrypt 4.x 遇超长密码直接抛错，导致注册崩溃。
# 已修（2026-07-04）：main.py 去掉 passlib，改用原生 bcrypt，
#       手动截断到 72 字节（hash_password / verify_password）。
#       生成仍是标准 $2b$ 哈希，老用户登录不受影响。
# 如果重装依赖又踩：别降级 passlib/bcrypt，确认代码走原生 bcrypt 即可。
# （依赖装在 ~/.venv，不是坏掉的 backend/venv）


 # 本地改完代码后
